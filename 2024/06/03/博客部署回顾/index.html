博客部署回顾

在经过三四天的折腾之后，总算完成了个人博客的初步搭建，包括本地部署Hexo，推送至个人 GitHub 仓库。

本地化部署 Hexo

下载git、node、hexo
创建博客路径:myblog
本地初始化hexo
hexo s:hexo本地启动命令
打开localhost:4000查看本地Hexo是否正常

推送GitHub

首先配置 sshkey
①进入~/.ssh
②执行ssh-keygen -t rsa -C “GitHub 绑定邮箱”
③cat rsa.pub 查看 sshkey
④进入 GitHub >set>配置 sshkey
测试 GitHub 的 ssh连通性
$$ ssh -T git@gitee.com
⑤在 myblog目录内修改_config.yml的发布（deploy ）配置
deploy :
#Deployment## Docs: https://hexo.io/docs/one-command-deploymentdeploy:
type: git
repo: git@github.com:GitHub 名称/ GitHub 仓库名称.git
branch: main
⑥推送到 GitHub的 git信息配置
git config –global user.name “yourname”
git config –global user.email “youremail”
⑦按照 deployer-git
npm install hexo-deployer-git –save
⑧将本地 blog 推送至 GitHub
sudo hexo c #清除缓存文件 db.json 和已生成的静态文件 public
sudo hexo g #生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)
sudo hexo d #自动生成网站静态文件，并部署到设定的仓库(hexo deploy 的缩写)
